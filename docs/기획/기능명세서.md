## 3. 기능 상세 명세

### SCR-001 생산통계 (`FR-001`)

| 항목 | 내용 |
|------|------|
| 접근 권한 | 관리자, 반장 |
| 갱신 주기 | 실시간 (1분 이내 자동 갱신) |

**출력 항목**

| 항목명 | 유형 | 설명 |
|--------|------|------|
| 간반 총 수량 | 숫자 | 당일 생성된 전체 간반 수 |
| 완료 수량 | 숫자 | 완료 처리된 간반 수 |
| 미완료 수량 | 숫자 | 대기/진행 상태 간반 수 |
| 생산 진행률 | % | 완료 수량 / 총 수량 × 100 |

**예외 처리**

| 상황 | 처리 방법 |
|------|---------|
| 당일 간반 없음 | "등록된 생산 데이터가 없습니다" 안내 표시 |

---

### SCR-002 현장관제 (`FR-002`)

| 항목 | 내용 |
|------|------|
| 접근 권한 | 관리자, 반장, 작업자 |
| 설치 위치 | 현장 모니터 5개소 |
| 갱신 주기 | 실시간 (작업 상태 변경 후 30초 이내 반영) |

**출력 항목**

| 항목명 | 유형 | 설명 |
|--------|------|------|
| 간반번호 | 텍스트 | 반별 개별 순번 |
| 작업 상태 | 상태값 | 대기 / 진행 / 완료 |
| 담당자 | 텍스트 | 배정된 작업자명 |
| 시작 시간 | 일시 | 작업 시작 시각 |
| 종료 시간 | 일시 | 작업 종료 시각 |
| 진행률 | % | 공정 진행 비율 |

**예외 처리**

| 상황 | 처리 방법 |
|------|---------|
| 디바이스 연결 끊김 | 마지막 데이터 유지 + 연결 오류 표시 |

### SCR-003 사무실관제 (`FR-003`)

| 항목 | 내용 |
|------|------|
| 접근 권한 | 관리자 |
| 화면 구성 | 생산 모니터 + 품질 모니터 2분할 |

**출력 항목**

| CCP 현재값 | 숫자 | CCP 기준값 대비 현재 측정값 |
| 이상 알림 | 상태 표시 | 기준값 초과 시 강조 표시 |

**예외 처리**

| 상황 | 처리 방법 |
|------|---------|
| 측정값 기준 범위 초과 | 이상 알림 팝업 + 담당자 알림 발송 (5분 이내) |
| 센서 연결 오류 | 연결 오류 표시 |

### SCR-004 주문 (`FR-004`)

| 항목 | 내용 |
|------|------|
| 접근 권한 | 관리자 |
| 연동 시스템 | WES |

**출력 항목**

| 항목명 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| 주문번호 | 텍스트 | O | WES 주문 식별자 |
| 제품코드 | 텍스트 | O | WES 제품 코드 |
| 제품명 | 텍스트 | O | 제품 이름 |
| 규격 | 텍스트 | X | 제품 규격 |
| 단위 | 텍스트 | O | 수량 단위 |
| 수량 | 숫자 | O | 주문 수량 |
| 채널 | 선택 | O | 쿠팡 / 컬리 / 자사몰 / 다파는 농부 / 스마트스토어 |
| 생산 순서 | 숫자 | O | 드래그 또는 직접 입력으로 조정 가능 |

**액션**

| 액션 | 설명 | 결과 |
|------|------|------|
| 주문 수신 | WES에서 주문 리스트 수동/자동 수신 | 주문 목록 갱신 |
| 순서 조정 | 생산 순서 변경 후 저장 | 즉시 반영 |
| 전개 이동 | 다음 단계(전개)로 진행 | SCR-005 이동 |

**예외 처리**

| 상황 | 처리 방법 |
|------|---------|
| WES 연동 실패 | 오류 메시지 표시, 수동 입력 허용 |

### SCR-031 BOP (`FR-031`)

> 2026.06.24 회의: 공정 연결 구조 설계 방향 구체화. 공정-간반 연계, 연결 확인, 저장/불러오기 기능 명세 추가.
> 2026.06.04 구현: 노드 에디터 형태로 전면 재설계. 상단 BOP 유형 탭, 좌측 공정 팔레트(그룹별), 우측 드래그-앤-드롭 노드 캔버스로 구성.

| 항목 | 내용 |
|------|------|
| 접근 권한 | 관리자 |
| 화면 구조 | 상단: BOP 유형 탭 / 좌측: 공정 팔레트 / 우측: 노드 캔버스 |

**기본 방향**
- 공정과 공정이 순서대로 연결되는 구조로 설계
- 연결되지 않은 공정이 있을 경우 사용자가 인지할 수 있어야 함
- 공정의 시작점과 종료점이 화면에 명확히 표시되어야 함

**입력 항목**

| 항목명 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| 공정코드 | 텍스트 | O | 중복 불가 |
| 공정명 | 텍스트 | O | |
| 공정 유형 | 선택 | O | 전처리 / 조리 / 냉각 / 포장 / 검수 / 출고 / 기타 |
| 순서 | 숫자 | O | 오름차순 정렬 기준 |
| 연결 공정 (다음) | 선택 | X | 순차 연결 형태. 없으면 종료점 |
| 작업 시간 | 숫자(분) | X | |
| 담당 설비 | 선택 | X | 설비현황에서 선택 |
| 간반 생성 여부 | 토글 | O | 이 공정에서 간반을 생성할지 여부 |

**주요 기능**

| 기능 | 설명 |
|------|------|
| 연결 확인 | "연결 확인" 버튼 클릭 시 끊어진 공정 탐지 및 결과 표시 |
| 시작/종료점 표시 | 리스트에서 시작 공정(아무도 지목하지 않는 공정)과 종료 공정(nextProcess 없음) 뱃지로 구분 표시 |
| 공정 그룹화 | 연결된 공정을 동일 공정 장소 기준으로 그룹화하여 표시 |
| 저장/불러오기 | 공정 프로세스 전체를 저장하고 재사용할 수 있는 기능 (명칭 추후 협의) |
| 순서 변경 | ↑/↓ 버튼으로 순서 변경, order 자동 재정렬 |

**공정-간반 연계 규칙**
- 간반 생성 여부가 Y인 공정만 간반이 생성됨
- 간반이 생성되는 경우 공정의 공간 정보와 간반의 공간 정보가 동일해야 함
- 일부 공정(검수, 중간 이동 등)은 간반 없이 운영 가능

**예외 처리**

| 상황 | 처리 방법 |
|------|---------|
| 연결되지 않은 공정 존재 | 연결 확인 버튼 클릭 시 해당 공정 코드 표시 |
| 순환 공정 설정 | 오류 메시지 표시 |
