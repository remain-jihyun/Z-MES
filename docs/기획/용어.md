## 2. 용어 목록

| 용어 | 영문 | 정의 |
|------|------|------|
| MES | Manufacturing Execution System | 제조 실행 시스템. 생산 계획부터 완료까지 현장 데이터를 실시간으로 수집·관리하는 시스템 |
| MIS | Management Information System | 경영 정보 시스템. 품목 DB 등 기준 데이터를 관리하는 상위 시스템 |
| WES | Warehouse Execution System | 창고 실행 시스템. 주문 데이터를 MES로 전달하는 외부 시스템 |
| Z-MIS | - | 집반찬연구소 내부 경영 정보 시스템. 품목 DB 연동 대상 |
| 간반 | Kanban | 반별로 생성되는 생산 작업 단위. 제품명, 수량, 담당 공정 등이 포함됨 |
| 전개 | - | WES 주문 코드를 현장 산 코드로 변환하고 수율을 적용해 소요량을 계산하는 과정 |
| 수율 | Yield | 투입 대비 산출 비율(%). 전처리·냉각·포장 3종으로 구분 관리 |
| BOM | Bill of Materials | 자재 명세서. 제품 생산에 필요한 원재료 구성과 소요량 기준 |
| BOP | Bill of Process | 공정 명세서. 생산 공정의 순서와 구조를 정의한 문서 |
| MRP | Material Requirements Planning | 자재 소요량 계획. BOM 기반으로 생산에 필요한 자재 수량을 자동 계산 |
| HACCP | Hazard Analysis Critical Control Point | 식품 위해요소 중점관리 기준. CCP(중요관리점) 온도·습도를 실시간 모니터링 |
| CCP | Critical Control Point | 중요관리점. HACCP에서 위해요소를 제어하기 위한 핵심 공정 지점 |
| NFC | Near Field Communication | 근거리 무선 통신. 태그 스캔으로 작업 시작·종료를 등록하는 데 사용 |
| TTO | Thermal Transfer Overprinter | 도미노 프린터 방식. 포장재에 소비기한 등 정보를 인쇄 |
| 로트 | Lot | 동일 조건으로 생산된 제품의 묶음 단위. 원재료부터 완제품까지 추적에 사용 |
| 선입선출 | FIFO (First In First Out) | 먼저 입고된 자재를 먼저 출고하는 재고 관리 원칙 |
| PULL 방식 | Pull Production | 주문 수요에 따라 생산을 시작하는 방식. 주문 → 전개 → 간반 → 생산 순으로 진행 |
| PUSH 방식 | Push Production | 계획에 따라 미리 생산하는 방식. 주간 생산계획 → 생산실행 순으로 진행 |
| 반 | - | 현장 작업 조직 단위. 간반은 반별로 생성·관리됨 |
| 산 코드 | - | 현장에서 사용하는 품목 코드. WES 코드와 구분됨 |
| 미출 | - | 계획 대비 생산량이 부족한 상태 |
| 과생산 | - | 계획 대비 생산량이 초과된 상태 |
| 추가불출 | - | 작업 중 재료 부족 시 추가로 자재를 요청하는 행위 |
| 안전재고 | Safety Stock | 수요 변동이나 납기 지연에 대비한 최소 재고량 |
| Z값 | Z-score | 통계적 서비스 수준에 따른 안전계수. 발주량 산출 공식에 사용 |

## 3. 단위 표기 규칙

> 2026.06.24 회의 결정사항. 데이터는 최소 단위(g / ml) 기준으로 저장하며, 화면 표시 시 아래 규칙을 적용한다.

### 3.1 표기 기준

| 범위 | 단위 | 표기 방식 | 예시 |
|------|------|----------|------|
| 1.0 이상 | kg / L | 소수점 첫째 자리까지 (둘째 자리 반올림) | 1.19kg → 1.2kg |
| 1.0 미만 | g / ml | g 또는 ml로 변환, 정수로 표기 | 0.1237kg → 124g |
| 1 ~ 999 | g / ml | 소수점 첫째 자리 반올림, 정수 | 123.7g → 124g |

### 3.2 화면별 표시 기준

| 화면 | 단위 표시 방식 |
|------|--------------|
| 현장 생산 화면 | kg / L 단위로 노출 |
| 간반(Kanban) 화면 | kg / L 와 g / ml 병행 노출 (검토 중) |
| DB 저장 | 항상 최소 단위(g / ml) 기준으로 저장 |
