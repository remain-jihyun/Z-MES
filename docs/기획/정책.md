## 2. 접근 권한 정책

> 권한 체계의 정의 및 관리 주체는 **Z-OS**이다. Z-MES 내 별도 권한 관리 기능은 없으며, OS 권한 체계를 그대로 사용한다. (2026.06.24 회의 확정)
> 이 섹션의 2.1~2.2는 Z-MES 화면 설계 기준 역할 구분이며, 2.3은 Z-OS 기준 권한 체계 전체를 정의한다.

### 2.1 Z-MES 화면 기준 역할 구분

Z-MES 메뉴 접근 권한 설계 시 사용하는 역할 분류이다. 실제 역할 부여 및 관리는 Z-OS에서 수행한다.

| 역할 | 직군 기준 | 설명 |
|------|----------|------|
| 관리자(사무직) | 사무직 | 생산계획 수립, 발주 승인, 마감 처리 전반을 담당 |
| 반장 | 현장직 | 간반 관리 및 현장 작업 지시 담당 |
| 작업자 | 현장직 | NFC 태그 스캔 및 생산 실행 입력 담당 |
| 시스템 관리자 | 사무직 | 디바이스 등록, 기초정보 관리 담당 |

### 2.3 회원 권한 체계 (Z-OS 기준)

모든 권한의 정의 및 관리는 Z-OS에서 수행한다. Z-MES, Z-MIS 등 분리된 시스템은 Z-OS의 권한을 연동하여 사용한다.

#### 권한 관리 중앙화 구조

```
Z-OS (중앙 권한 관리)
  └─ 직군 (사무직 / 현장직)
  └─ 역할 (마스터 / 관리자 / 일반)
  └─ 직책
  └─ 승인권한 (개인별)
  └─ 시스템별 메뉴 접근 권한 (부서별)
  └─ 부서별 품목 DB 제한 설정
       ↓ 연동
  Z-MES / Z-MIS
```

#### 직군 분류

| 직군 | 직책 구조 |
|------|----------|
| 사무직 | 대표이사, 임원, 본부장, 팀장, 파트장, 매니저 |
| 현장직 | 실장, 반장, 부반장, 사원 |

#### 역할(Role) 분류

| 역할 | 설명 | 권한 범위 |
|------|------|----------|
| 마스터 (슈퍼 관리자) | 시스템 전체에 접근 가능한 최상위 권한 | 회원 설정, 업무 프로세스 설정, 시스템 설정, 부서 설정, 보안 설정, 일반 유저 정보 설정 |
| 관리자 | 운영 관리 권한 | 회원 설정, 업무 프로세스 설정, 일반 유저 정보 설정 |
| 일반 | 제한적 접근 | 부서별 허용된 시스템 및 메뉴만 접근 가능 |

#### 승인 권한

승인 권한은 직책과 무관하게 회원 정보에서 개인별로 부여한다. 부여 및 회수는 Z-OS 관리자만 가능하며, 사무직 / 현장직 모두 동일한 방식으로 적용된다.

**운영 가이드라인 (승인권한 부여 권장 직책)**

| 직군 | 승인권한 부여 권장 직책 |
|------|----------------------|
| 사무직 | 팀장 이상 |
| 현장직 | 반장 이상 |

#### 시스템별 메뉴 접근 권한

부서별로 접근 가능한 시스템 및 하위 메뉴를 Z-OS에서 설정한다. Z-MES, Z-MIS 등 각 시스템의 메뉴 접근 권한도 Z-OS에서 통합 관리한다.

#### 부서별 품목 DB 제한 설정

부서별 / 컬럼(DB명) 단위로 접근 제한을 설정한다. 설정 주체는 Z-OS 관리자이다.

| 권한 | 설명 | 비고 |
|------|------|------|
| 보기 제한 | 해당 컬럼 조회 불가 | 설정 시 저장 제한 자동 적용 |
| 저장 제한 | 조회는 가능하나 저장(다운로드 포함) 불가 | 보기 제한 설정 시 비활성화 |

**운영 규칙**

- 신규 부서 생성 시 모든 DB 컬럼에 보기 제한 기본 적용
- 관리자가 필요한 컬럼만 제한 해제하는 방식으로 운영

#### 게시판 권한

게시판 권한 설정은 Z-OS에서 통합 관리한다. 권한 설정 주체는 Z-OS 관리자이며, Z-MES 등 각 시스템은 Z-OS 설정을 연동하여 적용한다.

| 권한 | 단위 | 설명 |
|------|------|------|
| 읽기 | 부서 단위 | 게시판 조회 권한 |
| 작성 | 부서 단위 | 게시글 작성 권한 |
| 승인 | 개인 단위 | Z-OS 승인권한 연동 |

### 2.4 Z-MES 권한 구현 현황

#### 현재 구조에서 이슈 없는 항목

아래 항목은 Z-OS에서 관리하므로 Z-MES 별도 구현이 불필요하다.

| 항목 | 처리 방식 |
|------|----------|
| 역할 기반 접근 제어 | Z-OS 연동으로 그대로 적용 가능 |
| 메뉴 접근 권한 | Z-OS에서 관리, Z-MES 별도 처리 불필요 |
| 품목 DB 제한 | Z-OS에서 관리, Z-MES 연동만 하면 됨 |
| 승인권한 연동 | Z-OS 개인별 승인권한 체크 연동으로 처리 가능 |
| 게시판 권한 설정 | Z-OS에서 통합 관리, Z-MES는 연동 적용만 하면 됨 |

#### 추가 구현 필요 항목 및 우선순위

| 우선순위 | 항목 | 내용 |
|---------|------|------|
| 🔴 즉시 | 업무 승인 프로세스 정의 | 어떤 업무에서 승인이 필요한지, 승인 단계 구조(1단계/다단계), 반려 시 재상신 프로세스, 승인권한자 부재 시 대리 승인 처리 방법 정의 필요 |
| 🔴 즉시 | 현장직 메뉴 접근 권한 확인 | Z-OS에서 현장직 대상 메뉴 접근 권한이 설정되어 있는지 확인. 미설정 시 Z-OS에서 명시적으로 설정 필요 |
| 🟡 단기 | 승인권한자 현황 조회 화면 | Z-OS에서 부여된 승인권한자 목록을 Z-MES에서 조회할 수 있도록 연동 필요 |
| 🟢 중기 | 권한 변경 이력 로그 | 권한 변경 시 변경자/변경 일시/변경 전후 내용 자동 기록 구조 필요. Z-OS 개발 항목 |

## 3. 데이터 정책

### 3.0 단위 저장 및 표시 정책

> 2026.06.24 회의 결정사항. 상세 표기 규칙은 용어 정의서 3절 참조.

**저장 원칙**: 모든 중량·용량 데이터는 **최소 단위(g / ml) 기준**으로 저장한다.

**화면 표시 원칙**

| 화면 유형 | 표시 단위 |
|----------|----------|
| 현장 생산 화면 | kg / L |
| 간반(Kanban) 화면 | kg / L 와 g / ml 병행 표시 (검토 중) |
| 관리자 입력 화면 | 입력 단위 자유, 저장 시 g / ml 변환 |

**반올림 규칙**

| 범위 | 단위 | 규칙 |
|------|------|------|
| 1.0 이상 | kg / L | 소수점 첫째 자리 (둘째 자리 반올림) |
| 1.0 미만 | g / ml | 소수점 첫째 자리 반올림 → 정수 |
| 1 ~ 999 | g / ml | 소수점 첫째 자리 반올림 → 정수 |
